サイト設計をしよう

いきなりコーディングを始めるのではなく、作るべきファイルやそれに付随するデータ、サイト全体のディレクトリ構成を設計しましょう。

そして、ディレクトリマップを作成して設計内容を見える化し、グループ内で共有しましょう。

ディレクトリマップに記載する内容(例)

  • ディレクトリ名とファイル名
  • ページの階層
  • URL
  • タイトル
  • ディスクリプション
  • コンテンツの内容
  • 担当者・進捗状況

HTMLファイルに限りませんが、ファイル名は原則として半角英数字のみで付けるようにしましょう。また、半角全角問わずスペースをファイル名に使用するのもやめましょう。もし、ファイル名に二語以上の単語を使用したいときはアンダースコア(_)やハイフン(-)でつなぎます。

HTML

HTMLの役割はドキュメントに意味づけをし、Webブラウザや検索エンジンのクローラーに正しく文書を理解させることです(=マシンリーダブル)。CSSでいくら美しくレイアウトしても、HTMLで構築する文書構造がめちゃくちゃでは意味がありません。

構文的に正しいHTML

HTMLには標準化団体によって規定されたルールがあります。そのルール通りにマークアップするのがWebコーダー・マークアップエンジニアの仕事です。

あなたが書いたHTMLが構文的に正しいか調べるには、ヴァリデーションツールを利用しましょう。

意味論的に正しいHTML

構文的に正しいHTMLを記述するだけでは、完璧なHTMLとは言えません。同時に意味論的に正しいHTMLの記述になっている必要があります。

使用している要素は文書の内容に対して適切ですか？付与された属性の種類や値は適切ですか？安易にdiv要素やp要素でマークアップするのはやめましょう。

画像

Web制作には様々な画像ファイルを使用しますが、いくつか注意点があります。

画像の形式は適切か

Web制作で使用する画像の形式として代表的なものはJPEG、PNG、GIF、SVG、WebPがあります。それぞれメリット・デメリットがありますので、利用シーンに応じて適切な形式で用意しましょう。

画像の解像度は適切か

Webページに表示しようとしている画像の解像度は適切な値になっていますか？Webでは一般的に72dpiに設定します。フリー素材サイトからダウンロードしたデータややデジカメで撮影したデータは300dpiや350dpiの画像であることがあります。事前に確認しましょう。

画像のサイズは適切か

ブラウザ上のレンダリングサイズを遥かに超える巨大な画像を利用していませんか？レンダリングサイズに合わせて適切にトリミング・リサイズをかけるようにしましょう。

近年では高精細ディスプレイが普及しており、レンダリングサイズの2倍や3倍の画像を要するケースも増えてきました。その場合はレスポンシブイメージというテクニックを使用すると良いでしょう。

CSS

コーディングで最も時間を費やすのはCSSに関する作業です。だからこそ、できるだけ効率よくシンプルに進めることが求められます。

いきなり書かない

「段取り八分、仕事二分」という言葉がありますが、コーディングはまさにこれです。いきなりVSCodeに向かう前にやるべきことがたくさんあります。

  • デザインカンプに目を通し、レイアウトで必要そうなボックスの目処をつける。
  • サイト全体で使いまわし出来そうなパーツ、レイアウトを洗い出す。
  • 各パーツのサイズ、余白、フォントサイズを確認したり、クラス名を考えるetc...

上記作業をデータではなく、手書きして手元に置いておくと良いです。

同じことを何度も書かない

CSSファイルが肥大化する原因の一つが、同じ記述の重複です。そもそも同じようなレイアウトや装飾は、最初に準備段階で洗い出してクラスを使い回すべきです。疑似クラス(:nth-child系など)やユーティリティクラスを利用すると記述を効率化できます。

詳細度(固有性)を揃える

CSSセレクタの詳細度(固有性)はできるだけ揃えるようにしましょう。行き当たりばったりで子孫セレクタや子セレクタを使っていると、予期せぬ箇所でスタイルが効かなくなったり、上書きできなくなったりします。

また、セレクタが異常に長くなるとサイトのパフォーマンスにも影響します。

個人的には、リセットやベースのセレクタは要素セレクタ(1点)や全称セレクタ、擬似クラス:where(0点)を、その他はクラスセレクタ1つ(10点)を基準に使用しています。

作業分担の方法

CSSを複数人で分担する際、最も考慮すべきことはセレクタの重複を避けることです。

一番簡単なのは、ページごとにbody要素に固有のid属性を付与し、セレクタの先頭にそのidを用いる方法です。

例えば、トップページのbody要素はid="top"とし、トップページに関するCSSのセレクタは#top .hogeという具合に書きます。メニューページも同様にbody要素にid="menu"とid属性を付与し、CSSのセレクタは#menu .hogeとします。このようにすれば、別々の人間が偶然.hogeというクラス名で異なるスタイルを定義したとしても、競合を避けることができます。

カスタムプロパティを使おう

CSSで変数のような役割を持つカスタムプロパティを有効に利用することで、作業効率は格段に向上します。

よく利用されるのはカラーコードを設定するケースですが、それ以外にもよく利用する余白の大きさやフォントサイズ、フォントファミリー、ヘッダーの高さ、角丸の数値、コンテンツ幅など、様々な値を設定できます。
